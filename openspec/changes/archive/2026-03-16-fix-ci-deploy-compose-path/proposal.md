## Why

The CI deploy step fails with "Could not find or access 'docker-compose.yml'" because the inline Ansible playbook uses a relative path (`src: docker-compose.yml`). Ansible resolves relative paths from the playbook directory (`/tmp/`), but `docker-compose.yml` lives in the repo checkout (`$GITHUB_WORKSPACE`). The file is never in `/tmp/`, so the copy task fails.

## What Changes

- Add a `compose_src` variable to the inline deploy playbook and use it in the copy task
- Pass `compose_src` from the workflow using `$GITHUB_WORKSPACE/docker-compose.yml` so Ansible finds the file
- Apply this fix to all three template CI workflows (go-service, node-service, python-service)

## Capabilities

### New Capabilities

- `template-ci-workflow`: CI/CD workflow generated by service templates; deploy step must correctly locate and copy docker-compose.yml from the workspace

### Modified Capabilities

- (none)

## Impact

- **Templates**: `.github/workflows/ci-cd.yml` in `templates/go-service`, `templates/node-service`, `templates/python-service`
- **Behavior**: Deploy step will succeed; no breaking changes
